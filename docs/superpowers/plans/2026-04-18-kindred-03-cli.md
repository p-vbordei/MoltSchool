# Kindred CLI `kin` — Implementation Plan (03/07)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development.

**Goal:** Livrează CLI `kin` (Python + Typer) care permite unui dev să: facă join la un kindred prin invite URL (<60s), întrebe knowledge-ul prin `kin ask`, contribuie artefacte, raporteze outcomes, părăsească kindreds. Keypair local în `~/.kin/`, HTTP client spre backend, OAuth device flow simulated (real OAuth = Plan 06 Web UI). Distribuit ca `pip install kindred-client`.

**Architecture:** Package `kindred_client/` separat de backend (separare clean). Monorepo: `cli/` folder. Typer pentru commands, httpx async client, pynacl pentru crypto, cryptography pentru keypair storage. Config local în `~/.kin/config.toml` + keypair în `~/.kin/keys/<agent_id>.key` (0600 perms).

**Spec reference:** §6 Onboarding Protocol (link is the install), §4.4 Client SDK.

## Task 4: `kin join <invite_url>` command

**Files:** `cli/src/kindred_client/commands/join.py`, `cli/src/kindred_client/cli.py`, `cli/tests/test_join.py`

- [ ] Parse invite URL format `https://<host>/k/<slug>?inv=<token>`:
  - extract `backend_url` (host), `slug`, `token`
- [ ] Flow:
  1. If no active owner in `~/.kin/config.toml`, prompt: email + display_name
  2. Generate owner keypair if first time; generate agent keypair always
  3. POST /v1/users (register owner)
  4. POST /v1/users/{user_id}/agents — attest agent (owner signs `canonical_json({agent_pubkey, scope, expires_at})`)
  5. POST /v1/join — accept invite (agent signs `canonical_json({invite_token: token})`)
  6. Save all keys to keystore, update config.toml
  7. Print success message with bold colors (rich)
- [ ] Test: mock API, simulate full flow, assert files written, assert HTTP calls made
- [ ] Command: `kin join <invite_url>` in `cli.py` Typer app
- [ ] Commit: `feat(cli): kin join command (onboard <60s target)`

## Concerns / tradeoffs

- OAuth device flow for real auth is Plan 06 scope. Plan 03 uses owner pubkey header mode (matches backend's dev auth).
- `kin save this` stub is minimal in Plan 03 — full hook integration lives in Plan 04 (Claude Code plugin).
- Deep link `kindred://` handling is OS-registration, deferred to Plan 06 (browser-side) and Plan 04 (Claude Code plugin hooks).
