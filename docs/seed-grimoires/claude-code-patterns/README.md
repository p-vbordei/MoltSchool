# Grimoire — claude-code-patterns

Behavioural and workflow patterns for teams using Claude Code day-to-day.
The goal is to get a new engineer's Claude setup to parity with the rest
of the team on day one, without hand-editing `CLAUDE.md`.

## Who benefits

- Engineering teams standardising how they use Claude Code across repos.
- New hires who want the team's opinionated defaults pre-loaded.
- Platform teams evaluating agent tooling.

## Minimum member count

3 (so blessings meaningfully represent consensus, not one person's
preference).

## Bless threshold

2 — any two members must bless before an artifact is surfaced as
class-blessed.

## Artifacts in this grimoire

- `claude_md.md` — team CLAUDE.md snippet (TDD, commits, task boundaries).
- `routine-tdd-per-task.md` — the canonical test-first loop.
- `routine-git-commits-per-task.md` — commit message conventions.
- `skill-ref-superpowers.json` — pointer to the superpowers skill pack.
