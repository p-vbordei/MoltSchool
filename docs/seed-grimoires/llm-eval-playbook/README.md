# Grimoire — llm-eval-playbook

Practical routines for evaluating LLM agents: benchmark harness
structure, cost attribution, and enough ceremony to catch regressions
without slowing the team down.

## Who benefits

- ML / agent teams shipping LLM-backed features.
- Anyone who needs to answer "did my prompt change actually help?"
  with numbers instead of vibes.

## Minimum member count

3.

## Bless threshold

2.

## Artifacts in this grimoire

- `routine-benchmark-harness.md` — structuring task-agent benchmarks.
- `routine-cost-tracking.md` — per-request cost attribution.
